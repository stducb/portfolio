INF3995 : Projet de conception d'un système informatique

Le projet visait à concevoir un système d'exploration automatique pour un essaim de drones miniatures. Pendant l'exploration, chaque drone devait envoyer sa position dans le but de creer une carte representant l'exploration des drones en temps reel. Pour atteindre cet objectif, un client Web a du être développé afin de communiquer avec les drones et un environnement de simulation. L'architecture generale du projet est la suivante:

  • Le serveur est le système central du projet. Il utilise le framework Flask pour gérer les requêtes et effectuer le traitement des données. Le serveur communique avec tous les autres systèmes. Il utilise MongoEngine pour échanger des données avec la base de données MongoDB en utilisant le protocole TCP/IP Sockets. Il communique avec les drones physiques à l'aide de la librairie cflib fournie par Bitcraze et la Crazyflie Radio PA en utilisant le protocole CRTP. Il communique également avec la simulation ARGoS par des sockets TCP et avec le client par le protocole HTTP.

  • Le client utilise ReactJS, pour créer l'interface utilisateur avec laquelle l'utilisateur interagit.

  • La simulation ARGoS est mise en œuvre en utilisant l'API et les modules fournis par ARGoS.

  • La partie embarquée est gérée en utilisant le firmware Crazyflie fourni par Bitcraze.

  • Afin d'acceder aux cartes crees par l'exploration des drones, nous utilisons une base de données MongoDB qui est hébergée sur le service MongoDB Atlas.

Dans le cadre de ce projet, j'ai eu l'opportunité de :
  • Programmer et simuler les drones Crazyflie pour qu’ils détectent leur environnement de façon autonome

  • Développer une page web pour permettre le contrôle des drones

  • Coordonner le projet dans une équipe de six personnes


Les principales technologies utilisées dans le développement du projet etaient Docker, MongoDB, Flask, React, Python, C, C++, Javascript, CSS.
